AlphaAgents Multi-Agent Research Platform

Mo Minoneshan
2026
GitHub Back to Project

0. Overview

AlphaAgents is a modular, production-grade multi-agent research platform that orchestrates LLM-powered specialist agents (fundamental analyst, valuation expert, sentiment analyst) to debate and synthesize equity research into actionable portfolio decisions. Inspired by the AlphaAgents paper, this implementation separates concerns across seven architectural layers: data ingestion, agent expertise, orchestration, backtesting, evaluation, human feedback, and application delivery.

Core Innovation: Rather than monolithic scoring, specialist agents argue about each stock via a debate mechanism, reaching consensus through a collaborator orchestrator while maintaining human-in-the-loop guardrails for risk management.

The system is designed for team collaboration: fundamental team owns the fundamental agent, sentiment team owns sentiment analysis, etc., with clean module boundaries and comprehensive testing.

1. Motivation

Modern quantitative equity research requires synthesizing multiple data streams:

  • Fundamental Analysis – Deep dive into financial statements, management quality, competitive position
  • Valuation Analysis – Technical factors, mean reversion, relative pricing
  • Sentiment Analysis – News narratives, market sentiment, consensus shifts
  • Macro Context – Economic cycles, sector momentum, cross-asset relationships

Traditional approaches use rigid scoring systems where signals are weighted and summed. This limits reasoning depth and makes it difficult to adjust weights in real-time based on market regime.

AlphaAgents solves this by having specialist agents collaborate through debate. Each agent brings expertise, supports its opinion with evidence from its data stream, and the collaborator orchestrates consensus. This approach:

  • Allows rich, multi-modal reasoning (text from filings, numerical technical indicators, news sentiment)
  • Enables easy weight adjustment: just re-prompt the collaborator
  • Provides audit trail: every decision includes agent rationales
  • Supports learning: human feedback logged for RLHF fine-tuning

2. System Architecture

The AlphaAgents system is built on seven architectural layers:

  1. Data Layer – Market connectors, filing downloader, news aggregator
  2. Agent Layer – Fundamental, valuation, sentiment specialists
  3. Utilities – Signal normalization, JSON parsing, cross-cutting helpers
  4. Analytics – Single-asset and portfolio backtesting, performance metrics
  5. Orchestration – Pipeline configuration and execution runners
  6. Human Feedback – Analyst review, override logging, RLHF capture
  7. Application Layer – CLI entry points, Streamlit dashboards, API servers

This separation ensures that teams can work on specialized components without breaking the overall pipeline. Data scientists improve sentiment analysis without touching backtesting logic.

3. The Seven Layers in Detail

3.1 Data Layer (src/data/)

Responsible for fetching and normalizing market data, SEC filings, and news.

  • market.py – OHLCV history via Yahoo Finance, aligned DataFrames
  • filings.py – SEC Edgar integration, 10-K/10-Q parsing, LLamaIndex document wrapping
  • news.py – News article aggregation, timestamp normalization, optional body hydration

3.2 Agent Layer (src/agents/)

Specialist agents that produce structured opinions (BUY/HOLD/SELL) from their data domains.

  • fundamental.py – RAG over SEC filings, produces fundamental conviction (Fast/Full mode)
  • valuation.py – Technical indicators + relative valuation, short-horizon signals
  • sentiment.py – Article summarization, recency-weighted sentiment aggregation
  • collaborator.py – Orchestrates debate, logs decisions, synthesizes consensus

3.3 Orchestration Layer (src/orchestration/)

Wires agents into repeatable workflows for single-ticker and multi-ticker analysis.

result = run_portfolio(
    tickers=["AAPL", "MSFT", "META"],
    lookback_days=45,
    fast_mode=False,
    include_sentiment=True
)

3.4 Analytics Layer (src/backtesting/ + src/evaluation/)

Converts agent signals into returns, computes performance metrics.

  • Single-asset and portfolio backtesting with rolling metrics
  • Sharpe ratio, drawdown, coverage analysis
  • Agent contribution attribution

3.5 Human Feedback Loop (src/feedback/)

Captures analyst overrides for RLHF experiments and audit trails.

3.6 Application Layer (apps/)

Entry points for running the system: CLIs, dashboards, batch pipelines.

4. Specialist Agents

4.1 Fundamental Agent

Reads SEC filings via LLamaIndex retrieval-augmented generation (RAG), prompts an LLM to form a structured opinion on business fundamentals.

Key Features:

  • RAG Pipeline: 10-K/10-Q cached as LLamaIndex documents, queried by the LLM
  • Fast Mode: Heuristic fallback (return None or neutral opinion) if LLM unavailable
  • LLM Selection: Configurable via FUNDAMENTAL_PROVIDER env (ollama/openai)
  • Output: Structured JSON with signal (BUY/HOLD/SELL) and rationale

4.2 Valuation Agent

Computes technical indicators and relative valuation metrics, produces a conviction signal on short-term price movements.

Key Metrics:

  • Momentum (RSI, MACD)
  • Mean reversion (Bollinger Bands)
  • Relative PE, PB, EV/EBITDA vs. peers
  • Profit margins, ROE, debt ratios

4.3 Sentiment Agent

Aggregates recent news articles, computes article-level sentiment (via LLM or heuristic), produces a recency-weighted narrative on market perception.

Output:

  • Numeric sentiment score (-1 to +1)
  • Key narrative themes extracted from top articles
  • Consensus summary (optional LLM synthesis)

4.4 Collaborator Agent

Takes specialist opinions and orchestrates debate. Prompts the LLM to weigh evidence from all three agents and produce a final consensus signal.

Output:

{
  "ticker": "AAPL",
  "final_signal": "BUY",
  "confidence": 0.78,
  "rationale": "Strong fundamental position offset by modest valuation concern, buoyed by positive sentiment shift",
  "agent_votes": {
    "fundamental": "BUY",
    "valuation": "HOLD",
    "sentiment": "BUY"
  },
  "position_weight": 0.22
}

5. Setup & Installation

5.1 Prerequisites

  • Python ≥ 3.11
  • Conda or Mamba for environment management
  • Optional: Ollama (for local LLM) or OpenAI API key

5.2 Installation

# Clone repository
git clone https://github.com/Minoneshan/DL_FE_01
cd DL_FE_01

# Create environment
conda env create -f environment.yml
conda activate ai_agents

# Install dependencies
pip install -r requirements.txt

# Optional: install Ollama
# brew install ollama (macOS)
# https://ollama.ai (Linux/Windows)

5.3 LLM Provider Setup

Option A: Ollama (Local, No Cost)

ollama pull mistral
export OLLAMA_MODEL=mistral
export FUNDAMENTAL_PROVIDER=ollama
export SENTIMENT_PROVIDER=ollama

Option B: OpenAI (Premium Quality)

export OPENAI_API_KEY="sk-..."
export OPENAI_MODEL="gpt-4o"
export FUNDAMENTAL_PROVIDER="openai"
export SENTIMENT_PROVIDER="openai"

6. Usage Patterns

6.1 Run the Portfolio Pipeline

# Zero-config mode (prompts "classic or alpha?")
python -m apps.strategy_runner.app

# Direct: Classic screening mode (20 candidates → 5-name portfolio)
python -m apps.pipeline.app \
  --strategy classic \
  --limit 20 \
  --target-count 5 \
  --period 3mo

# With guardrails (Alpha mode)
python -m apps.pipeline.app \
  --alpha-mode \
  --alpha-auto-approve \
  --limit 20 \
  --target-count 5

6.2 Analyze a Single Ticker

# Fundamental analysis
python -m apps.fundamental --ticker AAPL

# Valuation analysis
python -m apps.valuation --ticker AAPL

# Sentiment analysis
python -m apps.sentiment --ticker AAPL --days 7

6.3 Human-in-the-Loop Review

python -m apps.human_interface \
  --tickers AAPL MSFT META \
  --reviewer "Portfolio Manager" \
  --limit 3

Analyst is prompted to approve/reject each basket. Overrides logged to JSONL for reinforcement learning experiments.

6.4 Interactive Dashboard

streamlit run dashboard/app.py

Visualizes selected tickers, agent rationales, guardrail checks, and contribution to portfolio risk.

8. RAG Pipeline (Retrieval-Augmented Generation)

The fundamental agent uses LLamaIndex to build a vector index over SEC filings, enabling the LLM to retrieve and reason over relevant excerpts.

8.1 How It Works

  1. Download: Fetch 10-K/10-Q from SEC Edgar
  2. Parse: Extract structured tables and text sections
  3. Index: Create LLamaIndex document collection, auto-embed with sentence-transformers
  4. Query: Given a ticker, retrieve top-k relevant excerpts
  5. Prompt: LLM receives excerpts + question ("Is this a good investment?")

8.2 Performance

  • Typical latency: 2-5 seconds for full analysis (with OpenAI)
  • Token usage: ~2000-5000 tokens per filing analysis
  • Accuracy: LLM reasoning benefits from relevant excerpts (vs. full filing)

9. Backtesting & Evaluation

9.1 Single-Ticker Backtest

Given agent signals (BUY/HOLD/SELL), backtest module converts them to positions and computes returns.

  • Position Sizing: Equal weight, or confidence-weighted
  • Metrics: Sharpe, max drawdown, cumulative return
  • Output: Equity curve, signal timeline, metrics summary

9.2 Portfolio Backtest

Multi-ticker version that rebalances daily based on agent decisions.

  • Aggregates single-ticker signals into portfolio positions
  • Tracks portfolio-level metrics (Sharpe, correlation, contribution)
  • Supports guardrail constraints (max sector exposure, min diversification)

9.3 Guardrails

Automatic portfolio construction safety constraints:

  • Sharpe Threshold: Individual position must have Sharpe ≥ 0.10
  • Diversification: Min 5-10 holdings, max 25%/position
  • Sector Balance: Top 2 sectors max 40% of portfolio
  • Relaxed Mode: Allow exceptions if at least 50% of constituents pass

10. Human-in-the-Loop & RLHF

10.1 Feedback Capture

Every analyst override is logged as JSON with original signal, revised opinion, and reasoning. 

{
  "date": "2026-06-22",
  "ticker": "AAPL",
  "original_signal": "BUY",
  "override_signal": "HOLD",
  "reasoning": "Earnings beat priced in; risk/reward neutral",
  "agent_rationales": {
    "fundamental": "Strong moat, growing FCF",
    "valuation": "Fair value, limited margin of safety",
    "sentiment": "Positive on AI announcements"
  }
}

10.2 Using Feedback for RLHF

These feedback entries can be used to fine-tune the collaborator agent to better weight specialist opinions or to calibrate agent confidence.

  • Analyze which agents tend to disagree with humans → improve prompts
  • Fine-tune smaller models on feedback dataset
  • Measure Brier score improvement over time

12. Troubleshooting

12.1 LLM Not Responding

Issue: Timeout when calling fundamental or sentiment agent.

Solution:

  • Check Ollama is running: curl http://localhost:11434/api/tags
  • Or check OpenAI API key is set: echo $OPENAI_API_KEY
  • Increase timeout: export FUNDAMENTAL_TIMEOUT=120

12.2 Filing Download Fails

Issue: SEC Edgar returns 429 (rate limit).

Solution: Add delay between requests, or use cached local copies.

12.3 Portfolio Not Forming

Issue: Guardrails reject all candidates.

Solution: Enable --relaxed-guardrails to allow 50% compliance.

12.4 Dashboard Not Loading

Issue: Streamlit app crashes on startup.

Solution: First run pipeline with --export-pipeline to create dashboard data.
